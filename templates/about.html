À propos du projet

TCGDEX Explorer est une application web développée dans le cadre du projet Groupie Tracker

Présentation du projet

TCGDEX Explorer est une application web qui permet d'explorer les cartes du jeu de cartes à collectionner Pokémon en utilisant l'API publique TCGDEX. L'objectif est de fournir une interface intuitive permettant de rechercher des cartes, consulter des collections et gérer une liste de favoris.

Ce projet a été développé dans le cadre du cours "Groupie Tracker" à Ynov, avec pour consigne de créer une application web en Go qui exploite une API REST.

Go HTML CSS JavaScript API REST

FAQ - Gestion de projet

Comment avez-vous décomposé le projet ? Quelles ont été les phases clé ?

Le projet a été divisé en plusieurs phases distinctes :

  1. Analyse des besoins : Étude du sujet et identification des fonctionnalités essentielles (recherche, filtrage, pagination, favoris).
  2. Exploration de l'API : Recherche et test d'une API adaptée au projet. L'API TCGDEX a été choisie pour sa richesse en données et sa documentation claire.
  3. Conception de l'architecture : Organisation du code en packages (controllers, models, templates) et définition des structures de données.
  4. Prototypage : Création des wireframes pour visualiser l'interface utilisateur.
  5. Développement backend : Implémentation des routes, de la logique métier et de l'interaction avec l'API.
  6. Développement frontend : Création des templates HTML et des styles CSS pour l'interface utilisateur.
  7. Tests et débogage : Vérification du bon fonctionnement de l'application et correction des bugs.
  8. Documentation : Rédaction du README et de la documentation technique.
Comment avez-vous réparti les tâches entre les membres de l'équipe ?

Le projet étant individuel, j'ai dû assumer tous les rôles :

  • Architecte : Conception de la structure du projet et des modèles de données
  • Développeur backend : Implémentation des routes, de la logique métier et de l'interaction avec l'API
  • Développeur frontend : Création des templates HTML, du CSS et du JavaScript
  • Testeur : Tests manuels des fonctionnalités et correction des bugs
  • Documentaliste : Rédaction de la documentation technique

Pour organiser mon travail, j'ai utilisé une approche agile avec des sprints courts de 2-3 jours, chacun axé sur une fonctionnalité spécifique :

  • Sprint 1 : Mise en place de l'architecture et connexion à l'API
  • Sprint 2 : Implémentation de la recherche et des filtres
  • Sprint 3 : Développement de la pagination
  • Sprint 4 : Gestion des favoris
  • Sprint 5 : Finalisation de l'interface utilisateur et documentation
Comment avez-vous géré votre temps ? Avez-vous défini des priorités ?

La gestion du temps a été cruciale pour ce projet, surtout en travaillant seul. J'ai adopté les stratégies suivantes :

  • Priorisation des fonctionnalités : Les fonctionnalités ont été classées par ordre d'importance :
    • Priorité 1 : Connectivité à l'API et affichage de base des cartes
    • Priorité 2 : Système de recherche et filtres
    • Priorité 3 : Pagination et navigation
    • Priorité 4 : Gestion des favoris
    • Priorité 5 : Améliorations UI/UX
  • Planning quotidien : Utilisation de la technique Pomodoro (25 minutes de travail, 5 minutes de pause) pour maintenir la concentration
  • Estimation des tâches : Chaque fonctionnalité a été décomposée en tâches plus petites avec une estimation du temps nécessaire
  • Suivi de progression : Utilisation d'un tableau Trello pour visualiser l'avancement du projet
  • Marge de sécurité : Allocation de 20% du temps total pour la résolution de problèmes imprévus
Quelle stratégie avez-vous adoptée pour vous documenter ?

Pour mener à bien ce projet, j'ai consulté diverses sources d'information :

  • Documentation officielle :
    • Documentation de Go pour le développement backend
    • Documentation de l'API TCGDEX pour comprendre les endpoints et les données disponibles
    • Documentation de HTML/CSS/JavaScript pour le frontend
  • Ressources en ligne :
    • Tutoriels sur le développement web avec Go
    • Articles sur les bonnes pratiques de conception d'API
    • Forums et communautés comme Stack Overflow pour résoudre des problèmes spécifiques
  • Veille technologique :
    • Benchmarking d'applications similaires pour inspiration
    • Analyse de sites de cartes à collectionner pour comprendre les conventions d'interface
  • Organisation des connaissances :
    • Création d'une base de connaissances personnelle avec les informations importantes
    • Documentation des problèmes rencontrés et des solutions trouvées
